异步编程

异步编程的难点
  • 异常处理 
      var asyn = function (callback) {
      process.nextTick(callback)
  }
  //按照下面这样写是捕获不到的
  try{
      asyn(callback)
  }catch (e) {
      //TODO
  }

    node有个约定，如果在发生异常的情况下，将异常作为回调函数的第一个实参传回，如果为空值，则表明异步调用没有异常抛出，如果为空值，则 表明没有异常

      asyn(function(err, result){
  })

    还有另外一种情况：

      try{
      req.body = JSON.parse(buf, options.reviver)
      callback()
  }catch (err) {
      err.body = buf
      err.status = 400
      callback(err)
  }

    看上面的代码，如果没有发生异常，则正常；如果异常发生在callback，那么callback会被执行两遍；
    改写：

      try{
      req.body = JSON.parse(buf, options.reviver)
  }catch (err) {
      err.body = buf
      err.status = 400
      callback(err)
  }
  callback()

    在编写异步方法的时候，只要将异常正确传递给回调函数，不需要过多处理；
